# Notate 产品文档

**版本:** 1.0  
**日期:** 2025-01-11  
**状态:** 产品定义

---

## 一、产品定位

### 1.1 一句话描述

Notate 是一个**轻量级的个人知识 Layer**，通过快捷键随时捕捉想法，用 AI 自动整理，让你的信息可以被复用。

### 1.2 核心价值主张

> "你的想法终于有地方去了，而且能被复用。"

**我们不是帮你完成任务的工具，我们是帮你整理和使用信息的工具。**

### 1.3 与竞品的本质区别

| 维度 | Task-completion 工具（如阶跃AI） | **Notate** |
|------|----------------------------------|------------|
| 价值主张 | "帮你完成任务" | "帮你整理和使用你的信息" |
| Context 的角色 | 手段（执行任务的素材） | **目的**（Context 本身就是价值） |
| End Goal | 任务完成了 | 你的知识库变得更好、更可用 |
| 策略 | 宽（什么都想做，但做不好） | **窄但做得好** |

### 1.4 目标用户

- **B2C 个人用户**（不是 B2B）
- 有大量碎片化想法和信息需要整理的知识工作者
- 希望自己的 context 能够被复用，而不是每次都从零开始

### 2.1 产品形态

```
┌─────────────────────────────────────────────────────────────────┐
│                        Notate 架构                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   用户 80% 的时间                    用户 20% 的时间              │

│   │    Overlay      │               │      App        │         │
│   │  (快捷键唤起)    │               │   (主界面)      │         │

│   │  • Quick Capture│               │  • Home         │         │
│   │  • Msg Refine   │               │  • Library      │         │
│   │  • 快速搜索     │               │  • Workflows    │         │
│   └─────────────────┘               └─────────────────┘         │
│                                                                  │
│                         ↓  ↑                                     │
│              ┌──────────────────────────┐                       │
│              │      Knowledge Base       │                       │
```

### 2.2 核心理念

1. **Overlay 优先**：用户大部分时间通过快捷键交互，不需要打开完整 App
2. **渐进式知识库**：用户从随手记想法开始，逐渐积累成完整的个人知识库
3. **整理 > 执行**：我们的价值在于帮用户整理信息，而不是替用户完成任务
4. **窄但做得好**：专注于信息整理这一件事，做到极致

## 三、核心功能

### 3.1 Quick Capture（快速捕捉）

**入口**：全局快捷键（如 `⌘ + Shift + Space`）

**核心交互**：
```
用户按快捷键 → Overlay 出现 → 输入想法 → Enter → 消失
```

**设计原则**：
- 零摩擦：从想到记录 < 3 秒
- 不需要分类：先 dump 进来，AI 后续整理
- 即时消失：记完就走，不打断心流

**支持的输入类型**：
- 💭 Thought（想法/随记）
- 🔗 Link（链接）
- 📄 File（文件）

### 3.2 Knowledge Base（知识库）

**核心概念**：用户的所有 captures 汇聚成一个可搜索、可连接的个人知识库。

**渐进式构建路径**：
```
Day 1          Week 1           Month 1          Long-term
─────          ──────           ───────          ─────────
随手记想法      小笔记/摘录       拖入文件/PDF     完整知识库
一句话          几段文字          重要文档         + 自定义 workflow
```

**AI 自动处理**：
- 自动打标签
- 提取关键信息
- 关联相关内容
- 生成摘要

**知识库的展示方式（三种视图）**：

```

├─────────────────────────────────────────────────────────────────┤
```

2. **发现惊喜**：原来这两个想法是相关的！
3. **价值感知**："我真的积累了这么多知识"

### 3.3 知识库的 Output 机制

**核心问题**：知识库整理好了，怎么用起来？

```
┌─────────────────────────────────────────────────────────────────┐
│                   知识库的三种 Output 方式                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. Ask（问答式）                                                │
│     "我和 Sarah 上次聊了什么？" → AI 搜索并回答                   │
│                                                                  │
│  2. Compose（写作式）                                            │
│     "帮我写封邮件" → AI 用知识库内容生成草稿                       │
│                                                                  │
│  3. Surface（主动浮现）                                          │
│     你在写东西时，相关内容自动出现在旁边                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**Demo 价值**：展示知识库不是"死档案"，而是会主动帮助你的"活系统"。

#### 3.3.1 Ask（知识库问答）

**核心理念**：知识库不只是用来翻阅的，更是用来对话的。

**入口**：
- Overlay 中按 `/ask` 或点击 Ask 按钮
- App 主界面的 Ask 页面

**交互流程**：
```
用户提问: "我和 Sarah 上次聊了什么？"
           ↓
AI 搜索知识库，找到相关 captures
           ↓
AI 回答: "根据你 3 天前的记录，你和 Sarah 讨论了..."
           ↓
显示引用的原始 captures（可点击查看）
```

**使用场景**：
- "总结一下我这周所有关于 AI 的想法"
- "我之前看过一个关于 pricing 的视频，链接是什么？"
- "把我所有 #startup 标签的内容整理成一个大纲"

**设计要点**：
- 回答必须基于知识库内容，不是通用 AI
- 引用的来源要清晰可见
- 支持追问和深入对话

#### 3.3.2 Compose（写作辅助）

**核心理念**：写东西时，知识库应该主动帮忙。

**入口**：
- Overlay 中按 `/compose` 或点击 Compose 按钮

**两种模式**：

**模式 1: Write with Context（带知识库写作）**
```
用户: "帮我给投资人写一封更新邮件"
           ↓
AI 自动从知识库中找到相关内容：
  - 最近的产品进展
  - 关键指标
  - 团队更新
           ↓
AI 生成草稿，并标注哪些内容来自知识库
           ↓
用户编辑 → 复制使用
```

**模式 2: Find & Use（找到并使用）**
```
用户: "找一下我之前记的那个 pricing 策略"
           ↓
AI 找到相关内容
           ↓
用户选择 → 直接插入到正在写的内容中
```

#### 3.3.3 Surface（主动浮现 - 核心创新）

**核心理念**：不等你来问，相关内容自动出现。

**触发场景**：

**场景 1: Message Refinement 时**
```
用户截屏一封邮件，准备回复
           ↓
右侧 Context Panel 自动显示：
  - "你之前和这个人的沟通记录"
  - "相关的项目笔记"
  - "你答应过的 action items"
           ↓
用户可以选择引用这些内容
```

**场景 2: 查看某个 capture 时**
```
用户打开一条关于 "AI workflow" 的笔记
           ↓
Related Panel 自动显示：
  - 其他关于 AI 的想法
  - 看过的相关视频
  - 相关的论文笔记
           ↓
用户发现："原来我之前也想过这个！"
```

**场景 3: 新 capture 时**
```
用户输入一个新想法
           ↓
保存后，底部弹出：
  "💡 Related: 你 3 天前也记录过类似的想法"
           ↓
点击可以建立关联
```

**这是 "I didn't know I knew that" 的核心机制**。

**与 Message Refinement 的区别**：
| 维度 | Message Refinement | Compose |
|------|-------------------|---------|
| Context 来源 | 截屏/粘贴的外部内容 | 知识库内容 |
| 目的 | 润色已有内容 | 从零创作新内容 |
| 输出 | 优化后的消息 | 新的文档/邮件草稿 |

### 3.4 Habits（习惯）

**命名理念**：我们叫它 Habit 而不是 Workflow，因为这是你个人的、重复使用的信息处理习惯。

**类比**：类似 Claude 的 Commands —— 有默认的，也支持用户自定义。

**核心理念**：Habit 是**整理规则**，不是**任务执行**。

**Default Habits（内置）**：
- 收到 link → 自动提取标题/摘要，存入知识库
- 收到图片 → OCR + 存储
- 收到 PDF → 索引内容

**User-defined Habits（自定义）**：

用户用自然语言声明规则，例如：
```
"如果是 YouTube 链接，提取字幕，总结要点，加上合适的标签"

"如果是论文 PDF，提取 abstract 和 key findings，关联到相关的想法"

"每周日晚上，整理这周的所有 thoughts，生成一个 weekly reflection"
```

**智能推荐机制**：

当用户输入内容时，系统会：
1. 识别内容类型（链接 / 文本 / 图片 / 文件）
2. 细分识别（YouTube / Twitter / GitHub / 论文...）
3. 匹配用户定义的相关 Habits
4. 推荐最相关的 Habits 供用户选择

```
用户输入: https://youtube.com/watch?v=xxx
           ↓
识别结果: YouTube 链接
           ↓
推荐 Habits:
  → Summarize Video（提取字幕，生成要点摘要）
  → Save to Research（保存到 #research）
  → Quick Save（只保存链接）
```

### 3.5 Message Refinement（消息优化）

**场景**：用户在写邮件/消息时，需要根据上下文润色表达。

**Context 获取方式**：

**方式 1：截屏获取（推荐）**
```
Step 1: 快捷键唤起 Notate
        ↓
Step 2: 点击 [📷 Screenshot] 按钮
        ↓
Step 3: 进入截屏模式，框选需要作为上下文的区域
        ↓
Step 4: 截图自动作为 Context，回到输入界面
```

**方式 2：粘贴获取**
```
Step 1: 用户复制对话/邮件的上下文
        ↓
Step 2: 快捷键唤起 Notate → 粘贴 → Enter
```

**完整交互流程**：
```
Step 1: 获取 Context（截屏或粘贴）
        ↓
Step 2: 选择类型：[📇 Context] 或 [📝 Note]
        ↓
Step 3: 选择 Context 后，输入想要表达的内容
        ↓
Step 4: 选择 Style（可选）：Professional / Polite / Shorter / Longer
        ↓
Step 5: Enter → AI 生成优化后的消息 → 自动复制到剪贴板
        ↓
Step 6: 用户 Cmd+V 粘贴使用
```

**设计要点**：
- 截屏比复制粘贴更自然，可以捕获任何可见内容
- 分离"上下文"和"想说的话"
- Style 选择是可选的，有合理默认值
- 结果直接进剪贴板，一步到位

## 五、AI 的角色

### 5.1 两个角色

**角色 1: Workflow Designer & Implementer**
- 理解用户的自然语言声明
- 将声明转化为可执行的 workflow
- 在 workflow 中需要判断时做决策

**角色 2: Oracle（知识库的智能接口）**
- 回答关于用户知识库的问题
- 在写作时提供相关 context
- 连接不同时间点的 thoughts

### 5.2 AI 不做什么

- ❌ 不做通用任务执行（那是阶跃AI们在做的事）
- ❌ 不做复杂的多步骤自动化
- ❌ 不主动打扰用户

## 六、技术边界

### 6.1 当前阶段关注点

- Desktop App（Mac 优先）
- 快捷键全局唤起
- 本地知识库存储
- 基础 AI 处理（分类、标签、摘要）

### 6.2 不在当前阶段

- Mobile App
- 复杂的第三方集成
- Team/协作功能
- 云同步

## 七、成功指标

### 7.1 核心指标

| 指标 | 目标 | 说明 |
|------|------|------|
| Daily Captures | 10+ / 用户 | 用户是否养成了随手记的习惯 |
| Knowledge Reuse Rate | 30%+ | 多少 captures 被后续复用（搜索/引用） |
| Habit Adoption | 3+ / 用户 | 用户是否在定义自己的习惯 |
| DAU/MAU | 60%+ | 用户是否每天都在用 |

### 7.2 North Star Metric

**Knowledge Reuse Rate（知识复用率）**

> 用户过去记录的内容，有多少被后续搜索、引用、或者在 workflow 中被使用？

这个指标直接反映我们的核心价值：帮用户把信息整理好，让它们能被复用。

## 附录：与原 Notate 概念的关系

原 Notate 概念专注于 "quick capture + time tracking"。新定位保留了 quick capture 的核心，但：

1. **弱化 time tracking**：不再是核心功能
2. **强化知识库**：从 "记录" 升级为 "知识管理"
3. **增加 habits**：从 "被动记录" 升级为 "主动整理"
4. **增加 message refinement**：实际工作场景中的即时价值
5. **增加截屏获取 context**：更自然的上下文获取方式

核心理念一致：**轻量入口，随手记录，不打断心流**。
